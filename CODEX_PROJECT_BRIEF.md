# Codex プロジェクト概要: iOS ホーム画面アイコンアシスタント

## 0. プロジェクトの目的

iPhone のホーム画面に表示されるアプリ向けに、シンプルなカスタムアイコンを作成できる iOS アプリを構築する。

このアプリは、iOS ホーム画面の変更を完全自動化できると主張したり、そのような実装を試みたりしてはならない。代わりに、プライバシーを最優先にした半自動ワークフローを提供する。

1. ユーザーがホーム画面を手動で録画する。
2. ユーザーがその画面収録をこのアプリに取り込む。
3. アプリが動画をローカルで解析する。
4. アプリが表示されているアプリ名と、おおよそのホーム画面上の位置を検出する。
5. アプリが検出されたアプリ向けにシンプルなカスタムアイコンを生成する。
6. アプリがアイコンを書き出し、Apple ショートカットで「App を開く」ショートカットを手動作成する手順を案内する。

このアプリはシステムレベルのランチャーや MDM ツールではなく、あくまで**アシスタント**である。

## 1. iOS における絶対条件

実装全体を通して、Codex は以下の制約を守らなければならない。

### 1.1 インストール済みアプリの直接列挙を試みない

通常の App Store 向け iOS アプリは、端末にインストールされているすべてのアプリを自由に一覧取得できない。private API、SpringBoard へのアクセス、隠し URL スキーム、またはインストール済みアプリの監視に類する挙動は避けること。

許可されるアプローチ:

- ユーザーが提供したホーム画面録画からアプリを推定する。
- OCR とユーザー確認を利用する。
- 一般的なアプリ表示名のローカル辞書を任意で保持し、照合や補正に使う。

禁止されるアプローチ:

- Private framework の利用。
- 脱獄を前提とした設計。
- SpringBoard データベースの読み取り。
- 正確なインストール済みアプリ一覧を取得できると主張すること。

### 1.2 ホーム画面への自動配置を主張しない

このアプリは、アプリのアイコンやショートカットをユーザーのホーム画面へ無言で追加することはできない。最終的なホーム画面への追加は、ユーザー主導のショートカット操作でなければならない。

許可されるアプローチ:

- アイコン画像を生成する。
- アイコン用アセットを保存または書き出す。
- 手順を段階的に案内する。
- 可能であればショートカットアプリを開く、または説明を表示する。

禁止されるアプローチ:

- アプリが自動でアイコンを追加できるかのように装うこと。
- ショートカットアプリを操作するために不安定または private な自動化を使うこと。
- 「すべてのアイコンを自動インストール」のような誤解を招く UI。

### 1.3 録画はユーザー提供のみ

アプリがホーム画面のスワイプを制御したり、見えない形でホーム画面を録画したりしてはならない。

許可されるアプローチ:

- ユーザーに、コントロールセンターから iOS の画面収録を開始する方法を説明する。
- すべてのホーム画面ページをゆっくりスワイプするよう依頼する。
- 完成した動画を写真またはファイルから取り込めるようにする。

## 5. 中核となるユーザーフロー

### 5.1 オンボーディング

制約を明確に説明する:

- アプリはインストール済みアプリ一覧へアクセスしない。
- アプリが解析するのは、ユーザーが選択した録画のみ。
- 処理はデフォルトで端末上で行われるべきである。
- 検出されたアプリ名は、すべてユーザーが確認および編集する。
- ショートカット作成およびホーム画面追加は手動のままである。

主要 CTA:

- 「ホーム画面録画を取り込む」

補助 CTA:

- 「ホーム画面の録画方法」

### 5.2 録画手順画面

シンプルな手順を表示する:

1. コントロールセンターから画面収録を開始する。
2. 最初のホーム画面ページへ移動する。
3. すべてのページをゆっくりスワイプする。
4. 各ページで少し停止する。
5. 録画を停止する。
6. このアプリに戻って録画を取り込む。

品質向上のヒントも含める:

- 可能であれば標準のアイコンサイズを使う。
- 録画中は通知バナーが出ないようにする。
- 壁紙は見えていてよいが、OCR が失敗する場合は複雑すぎる壁紙を避ける。
- ゆっくりスワイプし、各ページで約 1 秒停止する。

アプリ自身がシステム全体の録画を開始するよう求めてはならない。

### 5.3 取り込み

対応内容:

- 写真から動画を取り込む `PhotosPicker`
- ファイルから動画を取り込む document picker

取り込み後:

- 必要な場合のみ、一時的なアプリストレージへ動画をコピーする。
- プライバシー通知を表示する。
- デフォルトで ON の「解析後に取り込んだ動画を削除する」オプションを提供する。

### 5.4 動画解析

パイプライン:

1. 動画からフレームを抽出する。
2. 安定して表示されているホーム画面ページを検出する。
3. 選択したフレームに OCR を実行する。
4. 認識テキストを正規化する。
5. OCR 結果をページ単位・グリッド位置単位でグループ化する。
6. 文字列を候補となるアプリ名に照合する。
7. 信頼度スコアを算出する。
8. レビュー UI を表示する。

### 5.5 検出アプリのレビュー

ユーザーが以下を行える必要がある:

- すべて確認する。
- 検出されたアプリ名を変更する。
- 誤検出を削除する。
- 不足しているアプリを手動で追加する。
- フォルダを別扱いでマークする。
- 感度を変えて再解析する。

各検出項目には次を表示する:

- アプリ名候補
- 信頼度
- ページ番号
- おおよそのグリッド位置
- 可能であれば、ラベルまたはアイコン周辺の切り出しスクリーンショット

### 5.6 アイコン生成

確認済みアプリごとに:

- 選択テーマを使ってシンプルなアイコンを生成する。
- デフォルトテーマはミニマルで権利侵害のないものにする。
- サードパーティ製アプリアイコンを直接コピーしてはならない。
- 文字の頭文字、汎用グリフ、抽象図形、またはユーザー選択シンボルを使う。

アイコン出力:

- PNG
- 1024x1024 のマスターアセット
- 任意で 180x180 のプレビュー
- 透過背景は任意。ただしホーム画面互換性のため、デフォルトは不透明背景とする。

### 5.7 エクスポートと案内

エクスポート方法:

- すべてのアイコンを写真に保存する。
- ZIP をファイルへ保存する。
- 個別アイコンを共有する。
- ショートカット設定用のチェックリストを生成する。

ユーザーへの案内:

1. ショートカットを開く。
2. 新しいショートカットを作成する。
3. 「App を開く」を追加する。
4. 対象アプリを選ぶ。
5. ホーム画面に追加する。
6. 生成したアイコン画像を選ぶ。
7. これを各アプリについて繰り返す。

これを完全自動であるかのように見せてはならない。

## 6. 動画解析の詳細

### 6.1 フレーム抽出

AVFoundation を使用する。

推奨実装:

- `AVAssetImageGenerator` を使う。
- 最初は低頻度、たとえば 1 秒あたり 2 フレームでサンプリングする。
- `appliesPreferredTrackTransform = true` を使う。
- 必要に応じて解析用にフレームを縮小する。
- 切り出し座標対応のため、元フレーム寸法は保持する。

擬似コード:

```swift
final class VideoFrameExtractor {
    func extractFrames(from url: URL, fps: Double) async throws -> [VideoFrameSample] {
        // AVAsset を読み込む
        // 動画の長さを判定する
        // CMTime のサンプル群を生成する
        // AVAssetImageGenerator.copyCGImage(at:actualTime:) を使う
        // timestamp + CGImage + size metadata を返す
    }
}
```

### 6.2 安定ページ検出

目的: スワイプ中フレームへの OCR 実行を避けること。

ヒューリスティック:

- 隣接フレーム間の画像差分を計算する。
- 複数サンプル連続で動きが少ないフレームは、安定表示されている可能性が高い。
- アイコンラベルが鮮明なフレームを優先する。
- 安定フレームを見た目の類似性でクラスタリングする。
- ホーム画面ページごとに代表フレームを 1 枚選ぶ。

実装案:

- フレームを小さなグレースケールサムネイルに変換する。例: 64x128。
- 平均絶対画素差を計算する。
- 差分が閾値未満の状態が続く区間を安定ウィンドウとしてマークする。
- perceptual hash またはサムネイル距離でクラスタリングする。

### 6.3 OCR

Vision の文字認識を使う。

推奨設定:

```swift
let request = VNRecognizeTextRequest()
request.recognitionLevel = .accurate
request.usesLanguageCorrection = true
request.recognitionLanguages = ["ja-JP", "en-US"]
```

注記:

- 日本語と英語のアプリ名に対応する。
- まずフルフレームで OCR を実行する。
- フルフレーム結果が弱い場合は、ラベルらしい領域への追加 OCR を検討する。
- レビュー UI 用にバウンディングボックスを保存する。

### 6.4 ラベル領域のフィルタリング

ホーム画面ラベルは通常アイコンの下に表示される。誤検出を減らすために:

- Dock アプリを意図的にサポートする場合を除き、ステータスバー付近やホームインジケータ付近の文字を無視する。
- グリッドの行らしい位置を検出する。
- アイコンらしい領域の下にある短いテキストブロックを優先する。
- 大きい文字、複数行、またはアイコングリッド外にあるウィジェット文字列は除外する。
- ラベル上に複数の小アイコンが並ぶ場合は、フォルダとして別検出する。

iPhone のレイアウトはモデルごとに異なるため、絶対座標のハードコードは避ける。正規化座標と端末サイズ別プロファイルを使うこと。

### 6.5 アプリ名の正規化

OCR 文字列を正規化する:

- 前後の空白を削除する。
- 改行を除去する。
- OCR で起きやすい誤認識を補正する。
- 全角と半角を正規化する。
- ラベルが省略表示されている場合は末尾の三点リーダを除去する。
- 表示用には元の文字列も保持する。

例:

```text
「設定 」 -> 「設定」
「App Store」 -> 「App Store」
「インスタ...」 -> ユーザー確認を求め、必要なら「Instagram」を候補提示する
「X」 -> アイコン切り出し等で確認できない限り低信頼度とする
```

### 6.6 マッチング

ローカル辞書を使用する:

```json
[
  { "displayNames": ["設定", "Settings"], "canonicalName": "Settings", "category": "system" },
  { "displayNames": ["写真", "Photos"], "canonicalName": "Photos", "category": "system" },
  { "displayNames": ["LINE"], "canonicalName": "LINE", "category": "social" }
]
```

マッチング戦略:

- まず完全一致を試す。
- ロケール考慮の大文字小文字非区別一致を行う。
- OCR ミスに対してあいまい一致を使う。
- OCR 信頼度 + 辞書一致 + ラベル位置に基づいて信頼度スコアを算出する。

辞書一致は、そのラベルがそのアプリである可能性を示すだけであり、アプリのインストールを証明するものではないことを示すこと。

## 7. アイコン生成要件

### 7.1 商標や既存アイコンのコピーを避ける

デフォルトではサードパーティ製アプリのロゴを再現してはならない。ジェネレーターはオリジナルでシンプルなアイコンを作成するべきである。

許可されるデフォルトスタイル:

- グラデーションまたは単色背景に頭文字を置く。
- 単線の汎用グリフ。
- カテゴリーベースの抽象図形。
- アプリ UI と書き出し画像レビューにおいてライセンスおよび利用条件が適切な場合の、ユーザー選択 SF Symbols。
- ミニマルな文字のみのアイコン。

避けるべきこと:

- Instagram、YouTube、LINE、X、TikTok、Gmail などのロゴをコピーすること。
- App Store のアイコンをスクレイピングすること。
- ブランドアイコンと紛らわしいほど似たアイコンを生成すること。

### 7.2 テーマモデル

```swift
struct IconTheme: Identifiable, Codable {
    let id: UUID
    var name: String
    var backgroundStyle: BackgroundStyle
    var foregroundStyle: ForegroundStyle
    var cornerRadiusRatio: CGFloat
    var fontName: String?
    var textMode: IconTextMode
}
```

### 7.3 生成アイコンモデル

```swift
struct GeneratedIcon: Identifiable {
    let id: UUID
    let appName: String
    let canonicalName: String?
    let imageURL: URL
    let themeID: UUID
    let createdAt: Date
}
```

### 7.4 レンダリング

1 つのレンダラー抽象化を使う:

```swift
protocol IconRendering {
    func renderIcon(for item: DetectedHomeScreenItem, theme: IconTheme, size: CGSize) throws -> UIImage
}
```

デフォルト出力サイズ:

- 1024x1024 PNG

プレビューサイズ:

- 180x180
- 120x120

## 8. プライバシー要件

デフォルトのプライバシー方針:

- 動画解析は端末上で行う。
- デフォルトではアップロードしない。
- ユーザーが明示的にオプトインしない限り、アプリ名を含む分析データを analytics に送らない。
- 取り込んだ動画は一時的なものとする。
- 「取り込んだ録画を削除する」操作を提供する。
- 「すべての解析データを削除する」操作を提供する。

機微情報のリスク:

ホーム画面録画には以下が含まれうる:

- 通知バナー
- 健康、金融、宗教、恋愛、仕事、または個人的関心を示唆するアプリ名
- 個人写真を含む壁紙
- カレンダー、位置情報、リマインダー、メッセージ、フィットネスデータを含むウィジェット

したがって:

- 取り込み前の警告を表示する。
- 可能であれば、解析前に動画のトリミングや切り抜きを行えるようにする。
- フル動画を保存するよりも、代表フレームだけを処理することを優先する。
- 解析後に保持するのは、派生したアプリ名候補と生成済みアイコン出力のみにする。

プライバシー文言:

> あなたの録画はこのデバイス上で解析されます。録画には通知、ウィジェット、壁紙、アプリ名などの個人情報が含まれる場合があります。録画と解析結果はいつでも削除できます。

## 9. App Store 審査安全性

やるべきこと:

- ユーザーが録画を取り込む方式であることを明確にする。
- Public framework のみを使用する。
- 最終的なショートカット設定はユーザー主導のままにする。
- 誤解を招く自動化表現を避ける。
- プライバシーポリシーを含める。
- インストール済みアプリ一覧を収集しない。

やってはならないこと:

- Private API を使うこと。
- SpringBoard の内部へアクセスすること。
- すべてのインストール済みアプリをスキャンすると主張すること。
- アイコンを自動インストールすると主張すること。
- 明示的な同意なしにホーム画面録画をアップロードすること。
- デフォルトでサードパーティロゴを再現すること。


### 10.4 DetectedHomeScreenItem

```swift
struct DetectedHomeScreenItem: Identifiable, Codable {
    let id: UUID
    var rawText: String
    var displayName: String
    var canonicalName: String?
    var confidence: Float
    var pageIndex: Int
    var gridPosition: GridPosition?
    var boundingBox: CGRect?
    var isFolder: Bool
    var userStatus: UserReviewStatus
}

enum UserReviewStatus: String, Codable {
    case unreviewed
    case confirmed
    case edited
    case rejected
}

struct GridPosition: Codable, Hashable {
    var row: Int
    var column: Int
    var region: HomeScreenRegion
}

enum HomeScreenRegion: String, Codable {
    case mainGrid
    case dock
    case widgetArea
    case unknown
}
```

```swift
    var pageIndex: Int
```

## 11. 推奨実装マイルストーン

### マイルストーン 1: 骨組みアプリ

- SwiftUI アプリを作成する。
- オンボーディング画面を追加する。
- 取り込み画面を追加する。
- 静的なモックレビュー用データを追加する。
- アイコンプレビュー画面を追加する。

受け入れ条件:

- アプリがビルドできる。
- ユーザーがモックデータで想定フローをたどれる。

### マイルストーン 2: 動画取り込みとフレーム抽出

- `PhotosPicker` / document picker を実装する。
- AVFoundation によるフレーム抽出を実装する。
- 抽出した代表フレームをデバッグ UI に表示する。

受け入れ条件:

- ユーザーが動画を取り込める。
- アプリがクラッシュせずにフレームを抽出できる。
- 大きな動画で進捗表示が出て、キャンセルできる。

### マイルストーン 3: OCR の概念実証

- 選択したフレームに Vision OCR を追加する。
- 認識したテキストのオーバーレイを表示する。
- 日本語と英語の認識をサポートする。

受け入れ条件:

- OCR 結果がバウンディングボックス付きで表示される。
- ユーザーがテキスト候補を確認できる。

### マイルストーン 4: 安定ページ検出

- フレーム差分による安定判定を実装する。
- フレームをページ単位にクラスタリングする。
- ページごとに代表フレームを 1 枚選ぶ。

受け入れ条件:

- アプリがスワイプ遷移中フレームを避けられる。
- 可能な限り、各ホーム画面ページが 1 回ずつ表現される。

### マイルストーン 5: アプリ候補抽出

- OCR 結果をアプリラベルらしいものに絞り込む。
- 名前の正規化とマッチングを行う。
- 信頼度スコアを追加する。
- レビュー UI を表示する。

受け入れ条件:

- ユーザーにもっともらしい検出アプリ一覧が表示される。
- ユーザーが編集、確認、却下、追加を行える。

### マイルストーン 6: アイコン生成

- デフォルトテーマを実装する。
- PNG アイコンをレンダリングする。
- プレビューを表示する。
- 一括生成をサポートする。

受け入れ条件:

- ユーザーが確認済みアプリ向けアイコンを生成できる。
- PNG ファイルがローカルに保存される。

### マイルストーン 7: エクスポート

- アイコンを写真に保存する。
- ZIP をファイルまたは共有シートへ書き出す。
- ショートカット設定チェックリストを追加する。

受け入れ条件:

- ユーザーが生成済みアイコンを書き出せる。
- ユーザーが明確な手動設定手順を受け取れる。

### マイルストーン 8: プライバシーとクリーンアップ

- 取り込んだ動画を削除する操作を追加する。
- 解析データを削除する操作を追加する。
- プライバシー画面を追加する。
- 明示的に後から導入しない限り、ネットワーク通信が発生しないことを保証する。

受け入れ条件:

- ユーザーが機微な取り込みデータを削除できる。
- アプリの挙動がプライバシー上の説明と一致する。

- 信頼度スコア算出

- 複数ページに同名アプリがある
- 省略されたアプリラベル

## 13. UI コピー草案

### 取り込み前警告

> ホーム画面の画面収録を選択してください。録画には通知、ウィジェット、壁紙、アプリ名などの個人情報が含まれる場合があります。解析はこのデバイス上で行われます。

### 解析結果の注記

> これらの結果は、録画内で見えている文字に基づく推定です。アイコンを生成する前に、必ず確認して修正してください。

### エクスポート時の注記

> iOS では、ホーム画面ショートカットはショートカットアプリで手動追加する必要があります。このアプリはアイコン画像を作成し、設定手順を案内します。

### 結果なし

> アプリ名をはっきり検出できませんでした。各ホーム画面ページで少し止まりながら、もう一度録画してみてください。

## 14. Codex 向け技術ガードレール

コード生成時に Codex が守るべきこと:

- Apple の public API のみを使う。
- `SpringBoardServices` のような private framework 名を避ける。
- インストール済みアプリ列挙を試みるコードを避ける。
- 主たる検出手段として隠し URL スキーム探索を使わない。
- OCR と動画処理はローカルに保つ。
- 長時間処理には async/await を使う。
- 解析にはキャンセル手段を提供する。
- UI 応答性を保つ。
- 可能な限り iPhone 固有の固定座標を避ける。
- 内部では正規化座標を使う。
- 生成アイコン画像はオリジナルかつ汎用的に保つ。

## 17. MVP の完了条件

MVP は、以下を満たしたときに完成とする:

- ユーザーがホーム画面録画を取り込める。
- アプリが代表フレームをサンプリングできる。
- アプリが端末上で OCR を実行できる。
- アプリが表示されているアプリ名候補一覧を提示できる。
- ユーザーが候補を確認、編集、却下できる。
- アプリがシンプルでオリジナルな PNG アイコンを生成できる。
- ユーザーがアイコンを書き出せる。
- アプリがショートカット設定手順を明確に案内できる。
- アプリがアイコンを自動インストールできると主張しない。
- アプリが取り込んだ録画と解析データを削除するためのプライバシー操作を備える。
